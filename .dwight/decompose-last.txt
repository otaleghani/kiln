**Content requirements:**
- Follow the EXACT same format as `docs/Features/Image Optimization.md` — YAML frontmatter with `title` and `description`, then heading, intro paragraph, and well-structured sections.
- `# Incremental Builds` heading
- Introductory paragraph: Kiln's `dev` command uses an incremental build pipeline to detect file changes and rebuild only what is necessary. Explain that this consists of four components working together.
- `## Architecture` section with a plain-text diagram or description of the pipeline:
  - **Filesystem Watcher** → detects raw filesystem events (write, create, remove, rename) using fsnotify
  - **Mtime Store** → tracks modification times, reports changed and removed files
  - **Dependency Graph** → maps wikilink relationships between notes (forward and reverse edges)
  - **Changeset Computation** → combines mtime changes with dependency graph to determine the full set of files to rebuild
- `## Filesystem Watcher` section:
  - Uses fsnotify to monitor the input directory recursively
  - Automatically watches new subdirectories as they are created
  - Skips dotfiles (`.obsidian`, `.git`) and `_hidden_` prefixed paths
  - Debounces rapid changes (default 300ms) to avoid redundant rebuilds
  - Multiple rapid saves collapse into a single rebuild
- `## Modification Time Store` section:
  - Tracks the last-known modification time of every file in the vault
  - On each rebuild trigger, walks the input directory and compares current mtimes against stored values
  - Reports two lists: **changed** files (new or modified) and **removed** files (previously tracked but no longer present)
  - Skips dotfiles and `_hidden_` paths, matching vault scan behavior
- `## Dependency Graph` section:
  - Built from the vault's wikilinks and markdown links after the initial scan
  - Maintains forward edges (source → targets it links to) and reverse edges (target → sources that link to it)
  - Parses both `[[wikilink]]` and `[text](path.md)` syntax
  - Strips aliases (`|`), anchors (`#`), and path prefixes to normalize target names
  - When a file is removed, its forward edges are cleaned up from the reverse index
- `## Changeset Computation` section:
  - Takes the changed/removed lists from the mtime store and the dependency graph
  - For each changed file, adds it and all its dependents (files that link TO it) to the rebuild set
  - For each removed file, adds its dependents to the rebuild set and the file itself to the remove set
  - Deduplicates: a file that appears as both changed and a dependent is only rebuilt once
  - Removed files are excluded from the rebuild set (no point rendering a deleted file)
  - Returns a `ChangeSet` with `Rebuild` (files to re-render) and `Remove` (files to delete from output)
- `## Lifecycle` section: Walk through what happens when you run `kiln dev`:
  1. Initial full build
  2. Mtime baseline populated (first scan sees all files as "new")
  3. Dependency graph built from vault wikilinks
  4. Watcher starts monitoring the input directory
  5. Server starts on `localhost:PORT`
  6. User edits a file → fsnotify event → debounce timer resets → timer fires → mtime update → changeset computed → rebuild triggered
- `## Current Limitations` section:
  - The rebuild currently performs a full site generation (the changeset is computed but not yet used to selectively rebuild individual pages)
  - The dependency graph is built once at startup and not updated after incremental changes
  - Only wikilink and markdown link dependencies are tracked; template or layout changes require a full restart
