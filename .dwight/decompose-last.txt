**Design Summary:**

The link preview feature requires:
1. A new JavaScript file (`link_preview.js`) that handles hover events on `a.internal-link` elements on desktop only
2. On hover, fetch the target page, extract the content from `#kiln-main`, and display it in a floating tooltip
3. CSS styles for the preview tooltip in `shared.css`
4. Wire the new JS file into the asset embedding pipeline (`assets.go`), layout loading (`layouts.go`), layout templates (`.templ`), and the build output step (`builder_default.go`)

The approach mirrors exactly how `search.js` is integrated: embedded asset → `text/template` parsed → written to output → `<script>` tag in layout templates.

Create the link preview JavaScript file at `/home/oliviero/Workspace/kiln/assets/link_preview.js`.

This file implements the full link preview feature. It should follow the exact same patterns as the existing JS files (`search.js`, `default_app.js`):
- Uses `var BASE_URL = "{{.BaseURL}}";` for the template variable (this gets replaced at build time, matching the pattern in `search.js`)
- Exposes `window.initLinkPreview` as an idempotent function
- Desktop-only: check `window.innerWidth >= 1024` or use `window.matchMedia("(hover: hover)")` — if the device doesn't support hover, skip initialization entirely
- Targets links with `a.internal-link` CSS class (this is the class set by the wikilink renderer in `internal/obsidian/markdown/wikilinks.go:452`)

Behavior:
1. On `mouseenter` of an `a.internal-link`, start a 300ms delay timer
2. On `mouseleave`, cancel the timer and hide the preview (with a small 100ms grace period so the user can move into the preview tooltip)
3. When the timer fires, fetch the href URL, parse the response HTML, extract the content from the `#content .content` element (this is the main prose div in both layouts)
4. Display the extracted content in a floating tooltip positioned near the hovered link
5. Cache fetched pages in a `Map` so repeat hovers don't re-fetch
6. The preview tooltip should:
   - Have id `link-preview-tooltip`
   - Be created once and reused (same pattern as lightbox in `default_app.js:261-277`)
   - Be positioned absolutely near the link (below if space, above if near bottom)
   - Have max-width 400px, max-height 300px, overflow-y auto
   - Support the preview itself being hovered (grace period on mouseleave)
   - Close on scroll, on Escape key, and on clicking outside
7. Handle edge cases: skip links with `target="_blank"`, skip anchor-only links (`#...`), skip the currently-active page URL
